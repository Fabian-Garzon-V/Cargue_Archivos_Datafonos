Especificación del Proyecto - Datafonos

Sistema de Gestión de Archivos y Consolidación de Datos en SQL Server

Descripción General del Proyecto

Este proyecto tiene como objetivo desarrollar un sistema que automatice la gestión, descompresión, consolidación y carga de archivos de datos hacia una base de datos SQL Server. La interfaz gráfica debe facilitar al usuario la carga de archivos, verificación de duplicados y parametrización de configuraciones para gestionar correctamente los archivos de entrada y el entorno de ejecución.

Objetivos Específicos

  • Descompresión de Archivos ZIP encriptados con contraseña.
  • Control de duplicados para evitar cargas repetidas en la base de datos.
  • Consolidación de Archivos y carga de datos en SQL Server.
  • Configuración personalizable a través de un archivo config.ini.
  • Interfaz gráfica amigable e intuitiva para el usuario final.

Especificaciones Técnicas

  • Lenguaje de Programación: Python 3
  • Framework de Interfaz Gráfica: Tkinter
  • Base de Datos: SQL Server
  • Archivo de Configuración: config.ini
  • Librerías Necesarias: pyodbc, configparser, tkinter, pyzipper, pandas, datetime, shutil, re

Requerimientos del Sistema

  • Sistema Operativo: Windows o Linux
  • Python 3.6 o superior
  • Controlador ODBC para SQL Server
  • Permisos de escritura y acceso a SQL Server

Arquitectura del Proyecto

El sistema está dividido en módulos independientes, cada uno enfocado en una funcionalidad principal: descompresión de archivos, consolidación y carga en base de datos, y administración de configuración. Los módulos están integrados en una interfaz gráfica común para facilitar la navegación.

Módulos y Funcionalidades

6.1 Módulo de Inicio de Sesión

Autentica al usuario contra la base de datos SQL Server y establece una conexión activa.

6.2 Módulo de Descompresión de Archivos

Permite al usuario seleccionar una carpeta y descomprimir archivos ZIP encriptados.

6.3 Módulo de Consolidación de Archivos

Procesa archivos descomprimidos, transforma y carga los datos en SQL Server, y controla duplicados.

6.4 Módulo de Modificación de Configuración

Permite al usuario ajustar parámetros de configuración a través de una interfaz amigable.

Flujo del Sistema

  1. Inicio de sesión y autenticación.
  2. Descompresión de archivos ZIP.
  3. Validación y transformación de datos.
  4. Carga de datos en la base de datos.
  5. Modificación de configuración.

Archivos y Configuración

El archivo config.ini almacena configuraciones necesarias como conexión a la base de datos y rutas de archivos. Ejemplo:

[database]
server = 192.168.2.106
database = PRUEBAS_BD
user = UsuarioBD

[paths]
decompression_origin = C:/ruta/descompresion
consolidation_origin = C:/ruta/consolidacion/origen
consolidation_dest = C:/ruta/consolidacion/destino

[decompression]
table_control_descompression = ControlCargaArchivosDatafonos

[consolidation]
table_control_cargue = ControlCargaArchivosDatafonos
table_datafonos = Datafonos

Pruebas y Validación

Se realizarán pruebas para verificar el funcionamiento del sistema en aspectos de autenticación, descompresión, consolidación y configuración.

  • Pruebas de autenticación: inicio de sesión con credenciales válidas e inválidas.
  • Pruebas de descompresión: manejo de archivos duplicados y archivos nuevos.
  • Pruebas de consolidación: manejo de duplicados y validación de campos de datos.
  • Pruebas de modificación de configuración: guardado de cambios en config.ini.

Conclusiones

Este sistema permite gestionar eficientemente el flujo de datos desde archivos ZIP hasta su consolidación en SQL Server, con un control adecuado de duplicados y configuraciones personalizables. La interfaz gráfica facilita el uso y operación del sistema, permitiendo una implementación exitosa en un entorno de producción.
